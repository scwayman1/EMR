Ontology Mapping Strategy

Leafly uses recreational tags ("Sleepy", "Happy"), but Verdant Apothecary uses clinical tags ("Insomnia", "Anxiolytic"). How should we translate these during the sync?

A

Static Code Mapping

Hardcode a dictionary in the codebase (e.g., "Sleepy" -> "Insomnia"). Fast to build, but requires a code deployment to update or add new terms.

B

Dynamic Database Mapping

Store the mapping rules in a Postgres table. Allows clinical admins to update the ontology translation on the fly without developer involvement.

C

AI-Assisted Tagging

Pass Leafly's raw strain description and effects through an LLM (like Claude) during sync to intelligently extract and assign our clinical therapeutic tags.
